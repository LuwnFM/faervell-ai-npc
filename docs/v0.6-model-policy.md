# v0.6 — strict OpenRouter model policy

This release removes the random `openrouter/free` router from production defaults.
Every model is now explicitly listed in `ACTOR_MODELS` or `PLANNER_MODELS`.

## Default order

Actor:

1. `nvidia/nemotron-3-super-120b-a12b:free`
2. `openai/gpt-oss-120b:free`
3. `nvidia/nemotron-3-ultra-550b-a55b:free`
4. `deepseek/deepseek-v4-flash`
5. `openai/gpt-oss-120b`
6. `mistralai/ministral-14b-2512`

Planner:

1. `deepseek/deepseek-v4-flash`
2. `openai/gpt-oss-120b:free`
3. `nvidia/nemotron-3-super-120b-a12b:free`
4. `openai/gpt-oss-120b`
5. `mistralai/ministral-14b-2512`

Actor replies still try free models first. Planner escalation starts with DeepSeek V4 Flash
because structured decisions benefit from its stronger reasoning; planner requests use
`reasoning.effort=high` while reasoning text remains excluded from Discord output.

## Hard blocks

The default blocklist rejects generic routers and known-bad candidates:

- `openrouter/free`
- `openrouter/auto`
- `openai/gpt-oss-20b`
- `nvidia/nemotron-nano-9b-v2`
- model slugs containing `laguna-2.1-xs` or `laguna-2-1-xs`

## Price ceiling

Every request sends an OpenRouter provider policy with these hard limits:

- prompt: at most `$0.20 / 1M` tokens;
- completion: at most `$0.20 / 1M` tokens;
- fixed request fee: `$0`.

DeepSeek V4 Flash currently has providers within this ceiling, so it is admitted without
raising the global limit. If no provider for a candidate model fits the ceiling, OpenRouter
falls through to the next explicit model. It cannot select a model outside the supplied arrays.

## Audit

Every successful and failed model call is recorded in `model_calls` and also written to
application logs with kind, selected model, candidates, token counts, cost and latency.
Reasoning text is excluded from the returned response.
