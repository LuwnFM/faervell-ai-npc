# Faervell AI-NPC 1.0.0 — Traveler Memory v2 rebuild

This release is a clean runtime rebuild, not another patch layer.

## Architecture

- PersonaUI Cortex is represented by immutable `identity_core` plus PostgreSQL
  digests for personal memory, relationship, open threads, testimony and shared
  world impressions.
- Mimir write/recall/persistence ideas are adapted into SQLAlchemy services:
  claims and evidence are separate from memories, deduplication is deterministic,
  and archive/restore/redact are lifecycle transitions instead of deletes.
- Context is rendered with the selected model's available token budget. There is
  no fixed Cortex character limit and no model/API call during memory maintenance.
- Testimony keeps speaker, subject, trust and disclosure policy. Repeated sources
  increase corroboration but never auto-confirm a rumor.
- The legacy `v07*`/`v080_*` installer is no longer used by runtime. Those files
  remain only as a rollback reference for one release.
- The public economy guide is indexed read-only (`144,768` positions, `24`
  states, `6,032` unique positions per state). `get_market_price` returns exact
  source-backed values or an explicit unknown; it never invents a price.
- Fandom API audit reaches the full namespace: `699` non-redirect articles
  (`689` readable in the audit; the live revision response currently yields
  `669`). The importer paginates continuation tokens and keeps a `669`
  minimum-document safety gate so transient API sparsity cannot replace the
  index with a partial crawl.
- Character sheets from channel `707461395209256982` are parsed into structured
  fields and refreshed every 48 hours; missing sheets are deactivated, not
  deleted. World-map metadata and transport modifiers are tracked under
  `data/world/`.
- Presence is a single global actor. PostgreSQL row locking plus a scene lease
  prevent simultaneous locations; queued summons wait until the current scene
  has completed its configurable quiet period.
- Every fresh process start resets the actor to channel `1488544832950374481`
  (the bureaucracy-domain startup lock). A GM releases it with
  `/stranger movement_lock enabled:false`; the release immediately performs a
  weighted roll over writable, enabled RP scenes. If no eligible scene exists,
  the actor remains in the startup channel and the lock stays released for the
  current process lifetime only.
- The supplied 500-template library is imported under
  `behavior-pack/template-library/`. It contains 32 quest archetypes and 500
  persona-audited examples: 370 directly usable and 130 guarded by a verified
  server action result. Rejected personality-incompatible records are excluded
  from runtime selection. Twelve additional operational identity templates
  cover the expected first-contact questions (name, origin, role, abilities,
  trade and purpose), bringing the local catalog to 512 records. Quest state changes are handled by the server-side
  `advance_quest` tool and require `server_verified=true`.
- Function coverage against baseline commit `ffac60a` is documented in
  `docs/v1.0.0-function-audit.md`: 339 of 347 callable names retained (97.69%).
  The only intentional callable-group replacement is the old memory monolith,
  now split into the Memory v2 package.
- The complete requirement matrix against the supplied integration PDF is in
  `docs/v1.0.0-pdf-compliance.md`.
- Compose resource caps target the production host (4 vCPU / 8 GiB RAM / 60 GiB
  NVMe): app 3 GiB, PostgreSQL 2 GiB, Redis 512 MiB, with the remaining budget
  reserved for Docker, backups and rollback.

## Rollout

1. Apply the additive schema migration and backfill hashes/claims in dry-run.
2. Start with `TRAVELER_MEMORY_V2_WRITE_ENABLED=true` and read disabled if needed.
3. Enable shadow recall, then enable read in the startup-lock channel.
4. Enable globally after attribution/privacy checks.

The 129 MB SQLite economy index is intentionally excluded from Git history. Upload
`data/economy/economy.sqlite3` to the same path on the production host before running
`scripts/deploy-production.sh`; the deploy script fails closed if the index is absent.

Rollback is the two read/write flags; new tables and data are retained.
