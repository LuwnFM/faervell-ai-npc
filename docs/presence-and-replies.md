# Присутствие Странника и адресация сообщений

## Как Странник понимает, что сообщение адресовано ему

В текущей локации ответ запускается только двумя способами:

1. прямое упоминание Discord-бота;
2. reply на сообщение Discord-бота.

Под последней частью RP-ответа может добавляться spoiler-подсказка. Она настраивается отдельно для каждой сцены:

```text
/stranger reply_hint enabled:true
/stranger reply_hint enabled:false
```

## Одна текущая локация

Все включённые `SceneConfig` являются возможными локациями, но `TravelerPresence` хранит только одну текущую. В остальных каналах Странник не отвечает и не изображает, будто физически находится сразу в нескольких местах.

Немедленное ручное перемещение GM:

```text
/stranger move_here
```

## Вероятности появления

Для каждой включённой сцены задаётся вероятность пройти отбор в очередной цикл движения:

```text
/stranger appearance_chance percent:20
```

Интервал цикла задаёт `TRAVELER_MOVEMENT_INTERVAL_SECONDS`. Если несколько локаций прошли вероятностный отбор, выбирается одна с учётом их значений вероятности.

При прибытии бот может опубликовать короткий RP-пост. Переключатель сцены:

```text
/stranger arrival_announcements enabled:true
```

## Пинг из другой локации

Пинг или reply в другой включённой сцене не вызывает мгновенный ответ. Локальный классификатор оценивает:

- содержательность сообщения;
- явную просьбу прийти или помочь;
- вопрос или обращение к Страннику;
- RP-формат;
- признаки тестового, пустого или OOC-пинга.

`RANDOM` только записывается в аудит. `AMBIGUOUS` и `INTENTIONAL` могут поставить эту сцену следующей целью. Приоритет более содержательного призыва может заменить прежнюю цель.

Глобальный переключатель:

```text
/stranger cross_location_summons enabled:true
/stranger cross_location_summons enabled:false
```

Очистка цели:

```text
/stranger travel_clear
```

Запланированный переход проверяется каждый цикл с вероятностью `TRAVELER_SUMMON_MOVE_CHANCE` и сохраняется до успешного перемещения либо ручной отмены.

## Reply на старый пост после ухода

Reply работает только для сообщений, отправленных во время текущего появления Странника. Если он ушёл из канала, старые сообщения остаются видимыми, но ответы на них:

- не вызывают RP-ответ;
- не ставят канал следующей целью;
- не возвращают Странника;
- только сохраняются в архиве разговора.

После повторного появления старые посты предыдущего визита также не открывают старый разговор. Для нового обращения нужно пингануть Странника в новом сообщении или ответить на пост текущего визита.
