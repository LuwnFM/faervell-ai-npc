# Соответствие PersonaUI Cortex / Mimir Memory Integration

Проверка выполнена по `PersonaUI Cortex and Mimir Memory Integration.pdf` перед выкладкой 1.0.0.

## Итог

Архитектура перенесена как адаптированный PostgreSQL/pgvector-аналог, а не как прямое подключение исходных фоновых модулей. В рабочем коде нет Markdown-Cortex, model tool-loop, отдельного memory-LLM, нейрохимии, фоновой жизни или автоматического переписывания `persona.md`.

## Осознанные отличия

1. Для production PostgreSQL используется настоящий FTS/pgvector; в локальном smoke/test окружении остаётся детерминированный hashing-embedder без сетевого вызова.
2. В `.env.example` V2 включён для готового релиза 1.0.0. Все флаги можно вернуть в `false` для shadow/staged rollout, не удаляя таблицы и данные.
3. `MemoryGraphService` не включается автоматически: это второй этап после стабилизации writer/claims/recall, как предписано PDF.
