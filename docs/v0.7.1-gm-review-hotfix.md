# v0.7.1 — GM review delivery hotfix

- Player-facing RP replies no longer mention GM, admins, tickets or approval workflows.
- Pending quest packets retain the internal review id so the Discord bot can post the request.
- Unposted pending reviews are retried on startup and immediately after `/stranger gm_channel`.
- Review-channel failures now include the channel id and Discord exception in logs.
- Approval/rejection notifications in the RP channel are written in-world.
