# Faervell AI NPC v0.7.4

## Knowledge retrieval, clean RP output and architecture synchronization

- `docs/architecture-source.md` synchronized with the current implementation, copied into the Docker image and registered as an internal restricted source.
- When the main wiki is already healthy but the architecture source is absent, startup imports only `architecture_source` instead of crawling the entire Fandom wiki again.
- The main Fandom import status is reported independently from auxiliary local sources, so an old missing local file no longer marks a successful 669-page wiki import as failed.
- Direct title retrieval uses proper-name terms from the original player query.
- Explicit entity types are treated as qualifiers joined with the proper name. Thus `Королевство Ивелтин` is preferred over `Республика Ивелтин`, while the generic word `королевство` can never select unrelated kingdoms by itself.
- Exact entity-page chunks are ranked above semantically similar pages such as `Эдемское Королевство`.
- Canonical citations remain in `ResponseBundle` and audit logs but are never appended to public RP messages.
- The actor is instructed not to print source lists, and the output guard rejects `Источники:` / `По источнику` service-style fragments.
- Regeneration confirmation is reduced to the private message `Ответ обновлён.` and no longer exposes model-routing reasons.
